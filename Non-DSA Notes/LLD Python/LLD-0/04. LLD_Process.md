## LLD INterviews :

- OOP Concepts, Design principles and patterns.

* How to approach the LLD round :
  - Overview
  - Gather Requirements
  - Class Diagram
  - Schema Diagram
  - Code
 
1) Get an overview of the system :
   - we already know the system : Align yourself with the interviewrs expectations
   - we dont know the system : Ask the interviewer to explain the system briefly.
   - 
   - Ask about what kind of system we need to design.
     - Entity (requirements + class diagrams)
     - working applciation : command line based or web based (rest api).
     - Do we need to store data in DB.
    
2) Gather Requirements :
   - 5-8 core features about the system
   - Try to visualise the system/app and suggest some of the important features. visualise either Physical structure or user journey
   - try to find out the edge cases. (clarify requirements)

3) class Diagram :
   - Classes/Interfaces
   - Relations between them
   - Design patterns

4) Schema Design
5) Code
