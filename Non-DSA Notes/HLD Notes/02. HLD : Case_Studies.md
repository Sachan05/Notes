* the first 3 case studies in L3 explains Caching can occur at: Browser, CDN, Application, Database layers.

1) Case Study - Scaler DSA Problem Submissions :
   - Problem :
     - Each submission needs large input/output files.
     - Fetching them from remote file storage takes ~2 seconds → slow submissions.
     
   - Explored Approaches
     - a) TTL-based Cache
       - Low TTL: frequent cache expiry → many misses → still slow
       - High TTL: stale data issue when files are updated
       - Not reliable for frequently changing files
       
     - b) Global Cache
       - Storing files in a single central cache causes:
         - Huge network overhead
         - Limited storage (if in-memory)
         - High eviction rate
         - Not suitable for large files
         
     - c) Local Cache
       - Fastest access (from local disk)
       - But updating a file across all servers is difficult
       - Invalidation issue
